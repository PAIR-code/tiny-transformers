TypeScript Functions & Escaped Values

Extend the power of the logic engine beyond static Algebraic Data Types. Learn how to wrap native TypeScript objects, define custom term operators, and execute dynamic calculations directly during logical transitions.

info_outline

1. Overview

Algebraic Data Types (ADTs) are excellent for modeling structured data, but some problems are better suited for traditional programming. For example, calculations like Peano arithmetic (using suc(suc(0))) can be extremely verbose and slow to reduce.

The logic engine provides two primary mechanisms to interface directly with TypeScript:

  • Escaped Values: Wrap arbitrary TypeScript/JavaScript instances (like numbers, strings, tensors, or models) and embed them directly inside logic terms.
  • TypeScript Functions: Register TS/JS functions in the logical context. These are executed dynamically during term evaluation, providing instant computations over terms.
integration_instructions

2. Escaped Values

An Escaped Value is a term that wraps a native TS/JS object. To define an escaped value, subclass EscapedValue and implement the required methods:

  • toString(): Defines how the escaped value displays as a string inside logic expressions and explorer tools.
  • equals(other): Defines the equality criteria used by the logic engine's type-checker and unification system.

            You can register a custom TypeScript function inside a logical Context using the defineTSFunction method.

            info
            

              
Evaluation Order

              Because the logic engine is Call-by-Value, all arguments passed to a TypeScript function are fully reduced to their normal form before the function callback is executed.
            

          


          

            warning
            

              
Equivalence Assumption (Canonicalization)

              TypeScript functions are expected to take logical terms / arguments and reduce/evaluate them into a canonical form (typically containing escaped values or base terms). 
              In this canonical form, type checking and unification rely on direct equality (i.e. the equals() method defined on your EscapedValue subclass).
              Therefore, you must ensure that your TS function always maps equivalent logical inputs to the exact same canonical output term so that direct value equality checks succeed during type verification.
            

          


          
Built-in Numerical Operations

          

            By default, every context registers a set of numerical helper functions via registerDefaultTSFunctions(). 
            These operations parse logic literal arguments as floats, perform calculations, and return a numeric literal term. 
            They are particularly useful for computing score weights inside simulator action rules:
          

          
    
            
  • add_num(?a, ?b): Returns a literal representing ?a + ?b.
    • 
            
  • sub_num(?a, ?b): Returns a literal representing max(0, ?a - ?b) (ensuring non-negative outputs).
    • 
            
  • mul_num(?a, ?b): Returns a literal representing ?a * ?b.
    • 
            
  • div_num(?a, ?b): Returns a literal representing ?a / ?b (returns 0.0 if division by zero occurs).
