Stochastic Generation & Story Simulation

Learn how to generate rich story distributions using the simulator. Run hundreds of logical state transitions stochastically, weight your actions with custom scores, and model complex population dynamics.

insights

1. Simulation Overview

While the manual state explorer is great for inspection and debugging step-by-step, modeling stochastic processes, emergent agent behaviors, or gameplay stories requires running many steps automatically.

The logic engine includes a probabilistic simulator that iteratively:

  • Finds all currently applicable actions and their resource matches.
  • Evaluates a numerical score for each possible action-match.
  • Converts the scores into a probability distribution.
  • Randomly samples an action based on those probabilities and applies the transition.

By looping through this lifecycle, we can simulate single trajectories or run Monte Carlo rollouts to observe the probability distribution of entire story paths.

2. Action Scores & Syntax

Actions are given weights/rates by annotating them with a score expression enclosed in square brackets immediately following the action's identifier:

action action_name [score_expression]: { LHS } -o { RHS };

If no score is provided, the engine defaults to a score of 1.0.

Evaluation Lifecycle of Scores:

  1. When the simulator identifies a match, it binds resources to logic variables (e.g., ?x).
  2. The variables inside the action's score expression are substituted with their bound values.
  3. The substituted term is fully evaluated (using Call-by-Value reduction) to a numeric literal.
  4. The resulting floating-point value is used as the raw score for that match. If the score evaluates to a negative number or is not a valid number, it defaults to 0.0.

3. Probability Modes

Once all applicable matches are scored, the simulator normalizes these scores into probabilities using one of two modes:

swap_calls
Proportional Mode

The probability of selecting a match is directly proportional to its score divided by the sum of all scores:

P(i) = score_i / Σ(score_j)

If the sum of all scores is zero, it defaults to a uniform distribution (every match has equal probability).

equalizer
Softmax Mode

Scores are mapped to probabilities using the Softmax function under a temperature parameter T:

P(i) = exp((score_i - max_score) / T) / Σ(exp((score_j - max_score) / T))

Temperature Control (T): A high temperature (e.g., T > 2.0) flattens the distribution, leading to more uniform/stochastic choice selection. A low temperature (e.g., T = 0.1) polarizes the probabilities, making the simulator almost always pick the action with the highest score.

6. Story Distributions & Plotting

        

        

          

            Running a simulation registers a timeline trace of resource counts. The simulator's Plot Mapping Configuration 
            tells the interface how to parse resources and map them onto a time-series line chart.
          

          

            A plot mapping rule is defined as a JSON configuration:
          

          
[
  {
    "name": "Rabbits Population",
    "literal": "rabbits",
    "argIndex": 0
  }
]
          
          

            This rule scans the multiset at each step, looks for all resources whose constructor is rabbits, 
            extracts their 0-indexed argument (which holds the count), parses it as a float, and sums it to plot the 
            total rabbit population line.
          

          

            This allows visualizing complex dynamics, such as predator-prey oscillations (Foxes & Rabbits preset example), 
            directly in the browser.
